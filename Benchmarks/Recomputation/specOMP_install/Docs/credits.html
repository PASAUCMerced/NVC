Creating a suite of source code benchmarks to provide a consistent, comparative measure of OpenMP and CPU-intensive performance is not simple. The challenges are numerous, for example:

  • locating meaningful programs;
  • ensuring that they port to numerous environments;
  • defining rules to govern how the programs are compiled, run and reported;
  • developing a toolset to help enforce these rules while still making things easy to use; and
  • defining a process by which decisions can be made amongst the group working on the project.

And in the end, you hope you have something that is fair, relevant, and understandable, that will address technology for a period of years.

Such is the case with SPEC OMP2012, which is an update and replacement for OMP2001. The new benchmark suite:

  • Provides a variety of applications and application areas included in the suite.
  • Uses application source code that is significantly complex.
  • Tests more features of OpenMP.
  • Provides for the optional collection of power data while running the benchmark.
  • Improves the tools for running and using the suite.
  • Clarifies and updates the run rules.
  • Improves documentation.

All of this is due to the efforts of a large team of people from around the world, that SPEC would like to thank (with apologies and thanks to those who may not be listed).
